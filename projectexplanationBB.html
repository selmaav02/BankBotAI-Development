Introduction

BankBot is an advanced AI chatbot designed to modernize customer support in the banking sector using Natural Language Processing (NLP).

The Transition to AI

Modern banking requires instant, reliable customer service. Traditional call centers are being rapidly augmented by Intelligent Virtual Assistants.

  • No Keywords Needed: Understands normal human sentences.
  • Always Online: Provides 24/7 uninterrupted assistance.
  • Scalable: Can serve thousands of customers at the exact same moment.

Why It Matters

Industry leaders like HDFC, ICICI, and SBI have successfully implemented chatbots to improve customer experiences.

  • Reduces average wait times from minutes to milliseconds.
  • Lowers operational costs for repetitive customer service queries.
  • Provides valuable insights through chat conversation analysis.

Understanding Rasa NLU

Rasa Natural Language Understanding (NLU) breaks down human text into structured data using two main concepts.

1. Intent Recognition

Identifying the purpose or goal behind the user's message.

"I want to block my debit card immediately."

Extracted Intent: block_card
"Check balance"check_balance
"Find an ATM"atm_locator

2. Entity Extraction

Pulling out specific pieces of information (parameters) from the message.

"What is the balance of my savings account?"

Extracted Entity:
Name: account_type | Value: savings

Confidence Scoring: Every prediction receives a percentage score. If the score falls below a set threshold, the bot intelligently asks for clarification.

Machine Learning Architecture

BankBot utilizes the Transformer Architecture — the same underlying technology that powers advanced language models like BERT and modern LLMs.

The DIET Classifier

Dual Intent and Entity Transformer (DIET) is a multi-task neural network.

  • Simultaneously predicts intents and entities to improve computational speed.
  • Uses Self-Attention Mechanisms to understand context. It looks at the whole sentence at once, not just word-by-word.
  • Trained specifically on a custom dataset of banking-related vocabulary.

How Text is Processed

  1. Tokenization: Sentence broken into individual words.
  2. Featurization: Words converted into mathematical vectors.
  3. Contextualization: Context is understood via attention spans.
  4. Classification: Model outputs final intent and extracted entities.

System Architecture

The system is built as a microservices-style architecture, allowing different parts of the application to operate and communicate efficiently.

💻
Presentation Layer
The Chat Interface for end-users and the Secure Dashboard for bank administrators.
⚙️
Logic & Web Layer
Flask server handling HTTP routing, APIs, session security, and data formatting.
🧠
Intelligence Layer
Rasa Server executing NLU tasks, plus the Action server for custom python logic.

Data Persistence Layer (SQLite)

Acts as the central memory bank — safely storing conversation logs, query metadata, and FAQs for offline analysis and dashboard reporting.

Web Interface & Backend Integration

Frontend (Client Side)

Built with pure HTML5, CSS3, and Vanilla JavaScript for ultra-fast load times.

  • Dynamic Chat UI: Features real-time typing indicators, auto-scrolling, and online status markers.
  • Admin Dashboard: Secure portal containing 6 distinct modules (Analytics, Settings, Logs).
  • Responsive: Fluid design adapting to mobile and desktop screens.

Backend (Flask Python)

The centralized hub routing all HTTP requests.

  • API Architecture: Serves JSON endpoints for dashboard statistics.
  • Security: Implements strictly enforced session boundaries and prevents unauthorized back-button navigation via cache headers.
  • File I/O: Generates CSV export files of query logs dynamically.

Secure Login Implementation:

Admin login uses server-side sessions. Every API endpoint enforces a strict session check before releasing sensitive analytics data.

Database Design

Using SQLite for lightweight, serverless, and robust data storage mechanism contained entirely within the project.

Table: query_logs

Captures every user interaction.

  • id: Primary Key
  • query: The raw text sent by user
  • intent: Categorized intent label
  • confidence: NLU decimal score
  • response: What the bot replied
  • timestamp: Exact Date & Time
Table: faqs

Manages Frequently Asked Questions.

  • id: Primary Key
  • question: Question text formulation
  • answer: Detailed text response
  • created_at: Auto timestamp formulation
